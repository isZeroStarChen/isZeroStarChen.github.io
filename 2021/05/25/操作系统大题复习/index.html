9、在建立一个操作系统时有几个设计目的，例如资源利用、及时性、健壮性等。请列举俩个可能相互矛盾的设计目的。

答：

​ ①并发性和高效性。一方面系统希望进程并发进行，另一方面，面对有限的资源有需要控制总线的优先权，使其及时完成，达到最大效率。

​ ②用windows系统来体现一个矛盾：管理员权限。尽管有的时候运用管理员权限对系统进行删除、改写等操作，但还是会被提醒说权限不够

15、假设一个计算机系统有高速缓存、内存(RAM)以及磁盘，操作系统用虚拟内存。读取缓存中的一个词需要1ns，RAM需要10ns，磁盘需要10ms。如果缓存的命中率是95%，内存的命中率是99%(缓存失效时)，读取一个词的平均时间是多少？

答：

​ 0.95x1+0.05x0.99x10+0.05x0.01x10x106=5001.445 ns

23、有一个文件，其文件描述符是fd，内含下列字节序列：3，1，4，1，5，9，2，6，5，3，5。有如下系统调用：

lseek(fd,3,SEEK_SET);

read(fd,&buffer,4):

其中lseek调用寻找文件中的字节3。在读操作完成之后buffer中的内容是什么？

解析：

​ lseek函数将读写位置指向fd文件头后，增加了3个位移量至第四个字节处。用read函数从fd读取4个字节存入缓冲区。

答：

​ 1，5，9，2

6、一个计算机的RAM有4GB，其中操作系统占512MB.所有进程都占256MB(为了简化计算)并且特征相同。要是CPU利用率达到99%，最大I/O等待是多少？

答：

​ 4GB=4x210=4096MB 最大进程总容量：4096-512=3584MB 最多进程数：3584/256=14

设最大I/O等待是x，则有1-x14>99%，解得x<72%

答：

​ 最大优点是：用户级线程可以在不支持线程的操作系统上实现，效率快

​ 最大缺点是：如果一个阻塞，那就全部阻塞。

45、有5个批处理作业A~E，它们几乎同时到达一个计算中心。估计它们的运行时间分别为10、6、2、4和8分钟。其优先级(由外部设定)分别为3、5、2、1和4，其中5为最高优先级。对于下列每种调度算法，计算其平均进程周转时间，可忽略进程切换的开销。

​ (a)轮转法

​ (b)优先级调度

​ (c)先来先服务(按照10、6、2、4、8次序进行)

​ (d)最短作业优先

解析：

​ 轮转法：用批处理作业最少所花费的时间乘以批处理作业的个数，得到每步轮转所需要花费的时间。在完成每步完成轮转后，重新选取最少花费时间乘以剩余批处理作业时间个数，进行计算。

​ 优先级调度：根据优先级进行计算，排在下一优先级要加上一优先级所花费的时间

​ 先来先服务：按照题目所给的排列顺序直接进行计算

​ 最短作业优先：按照批处理作业时间从低到高排列进行计算。

答：

​ (a) (10+18+24+28+30)/5=22

​ (b) (6+14+24+26+30)/5=20

​ (c) (10+16+18+22+30)/5=19.2

​ (d)(2+6+12+20+30)/5=14

4、在一个交换系统中，按内存地址排列的空闲区大小是10MB、4MB、20MB、18MB、7MB、9MB、12MB和15MB。对于连续段的请求：

​ (a)12MB

​ (b)10MB

​ (c)9MB

使用首次适配算法，将找出哪个空闲区？使用最佳适配、最差适配和下次适配算法呢？

解析：

​ 首次适配算法是按照题目所给空闲区顺序，从头找到最近的大于等于请求数的空闲区，如果找到，又再次从头开始

​ 最佳适配算法找最接近空闲区大小的空闲区

​ 最差适配算法找最大的空闲区，然后用最大的空闲区减去所需要的连续段请求，得到一个新的空闲区空间，再重新排序，以此类推。

​ 下次适配算法找是按照题目所给空闲区顺序，从头找到最近的大于等于请求数的空闲区，如果找到，就继续从当前位置开始

答：

​ 首次适配算法：找出20MB、10MB、18MB

​ 最佳适配算法：找出12MB、10MB、9MB

​ 最差适配算法：找出20MB、18MB、15MB

​ 下次适配算法：找出20MB、18MB、9MB

36、一个计算机有4个页框，载入时间、最近一次访问时间和每个页的R位和M位如下图所示(时间以一个时钟周期为单位)：

页面 载入时间 最近一次访问时间 R M
0 126 280 1 0
1 230 265 0 1
2 140 270 0 0
3 110 285 1 1

(a)NRU算法将置换哪个页面？

(b)FIFO算法将置换哪个页面？

(c)LRU算法将置换哪个页面？

(d)第二次机会算法将置换哪个页面？

解析：

​ 常见页面置换算法如下：

​ NRU(最近未使用算法)是根据R、M状态把页面分成四类，把编号最小的类随机选择一个页面置换。

​ FIFO(先进先出页面置换算法)是先进入的页面就先换出，即淘汰最老的页面。

​ LRU(最近最久未使用算法)是将最久未使用的页面置换出去，但当有中断时，造成的该元素就算已经使用过。

​ 第二次机会算法是先检查该页面是否被使用，如果未被使用，即R、M=0，就替换该页面，如果在被使用，就保留该页面。

36、考虑图4-21背后的思想，目前磁盘平均寻道时间为6ms，旋转速率为15000rpm,每道为1048576字节。对于大小各位1KB、2KB、4KB的磁盘块，传送速率各是多少？

解析：读取一个k字节的块所需要的时间t=寻道时间+旋转延迟+传送时间。

​ 传送速率(单个字节)v=k/t

答：

​ 旋转周期 = 1 / 15000 (min) = 1 / 15000 x 60 (s)=1 / 15000 x 60 x 103 ms= 4 ms

​ 旋转延迟 = 4 ms x 1/2 = 2 ms

​ 假设传送k个字节，则有传送时间 = (k / 1048576 ) x 4 ms

​ 时间t = 寻道时间+旋转延迟+传送时间=6ms+2ms+ (k / 1048576 ) x 4

​ k = 1KB = 210 byte(字节) ,t = 8.004, v = k/t= 127.936

​ k = 2KB = 211 byte(字节) ,t = 8.008, v = k/t= 255.744

​ k = 4KB = 212 byte(字节) ,t = 8.016, v = k/t= 510.978

40、一个UNIX系统使用4KB的磁盘块和4字节磁盘地址。如果每个i节点中有10个直接表项以及一个一次间接块、一个二次间接块和一个三次间接块，那么文件最大为多大？

解析：

​ 用磁盘块除以磁盘地址的结果2n，其中n为指针个数，那么一次间接块就有2n个指针，二次间接块就有22n，三次间接块就有23n个指针。磁盘块个数=一次间接块指针个数+二次间接块指针个数+三次间接块指针个数+直接表项。文件大小 = 磁盘块个数x磁盘块大小。

答：

​ 4KB/4byte = 210，所以i结点有10个指针。一次间接块有210个指针，二次间接块有220个指针，三次间接块有230个指针。

​ 磁盘块个数 10 + 210+220+230 = 1074791434

​ 文件大小 = 磁盘块个数x磁盘块大小=(10 + 210+220+230)x4KB=1074791434x4KB=4.004TB=4TB

28、考虑一个包含16个磁头和400个柱面的磁盘。该磁盘分成4个100柱面的区域，不同的区域分别包含160个、200个、240个和280个扇区。假设每个扇区包含512字节，相邻柱面间的平均寻道时间为1ms，并且磁盘转速为7200rpm。计算磁盘容量，最优磁道斜进以及最大数据传输率。

解析：

​ 存储容量 = 磁头数 x 磁道(柱面)数 x 每道(柱)扇区数 x 每扇区字节数

​ 计算最优磁道时，先计算在平均寻道时间内，磁盘转过的角度，再用磁盘转过的大小乘以每个区域的扇区数。

​ 最大数据传输率是最外层区域内的磁头正在读取/写入时获得的速率。= 扇区数 x 每扇区字节数 x 磁盘转速

答：

​ 磁盘容量 = 16x100x(160+200+240+280)x512=720896000

​ 7200rpm = 120 r/s。在1ms的平均寻道时间内，磁盘转过的角度为10-3x120=0.12。

所以区域1转过：0.12x160=19.2个扇区，区域2转过：0.12x200=24.0个扇区，区域3转过：0.12x240=28.8个扇区，区域4转过：0.12x280=33.6个扇区。所以对于4个区域柱面来说，最优磁道斜进分别为19.2、24.0、28.8、33.6。

​ 最大数据传输率 = 280x512x120=17203200 byte/s = 16.4025 MB/s
